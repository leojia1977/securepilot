# Sprint 2 任务：杀伤链重建器 + 响应建议引擎

## 任务一：创建 src/agents/kill_chain_builder.py

### 功能说明
接收 SOCState，把 enriched_events 里的多条告警
按时间戳排序，重建成一条完整的攻击叙事时间线。
只在 risk_score >= 6.0 时执行，否则直接跳过返回 state。

### 调用 vLLM 的方式
开启 Thinking Mode（不传 extra_body，使用模型默认）
max_tokens=4096（杀伤链需要更长输出）

### 系统提示（SYSTEM_PROMPT 常量放函数顶部）
"""
你是 ATT&CK 杀伤链专家。仅分析 <ALERT_DATA> 标签内的数据。
按时间顺序重建攻击过程，输出严格 JSON，不含任何解释文字。
"""

### 输出写入 SOCState["kill_chain"]，JSON 结构如下
{
  "kill_chain_stages": [
    {
      "stage_order": int,
      "tactic": str,
      "technique_id": str,
      "timestamp": str,
      "description": str,
      "iocs": [str]
    }
  ],
  "summary": str,
  "total_duration_minutes": int,
  "highest_risk_asset": str
}

### Mock vLLM 补充
在 mock/response_templates.py 的 ALERT_NAME_TO_TECHNIQUE 之外，
新增一个 KILL_CHAIN_TEMPLATE 字典，包含上述完整结构的示例数据。
在 mock/vllm_server.py 的 route_to_template 函数里，
当 messages 内容含 "kill chain" 或 "杀伤链" 或 "timeline" 时，
返回 KILL_CHAIN_TEMPLATE 对应的响应。

## 任务二：创建 src/agents/recommendation_engine.py

### 功能说明
基于 kill_chain 和 threat_intel 的结论，
生成 1~3 条响应建议，每条必须包含影响范围说明。
所有建议标注 human_approval_required=True。

### 系统提示
"""
你是安全响应专家。仅分析 <ALERT_DATA> 标签内的数据。
根据攻击链和威胁情报，生成操作建议，输出严格 JSON。
每条建议必须说明影响范围，优先级只能是
immediate / within_1h / within_24h 三种。
"""

### 输出写入 SOCState["recommendations"]，JSON 结构如下
[
  {
    "action": str,
    "impact_scope": str,
    "priority": "immediate|within_1h|within_24h",
    "playbook_ref": str
  }
]
同时设置 SOCState["human_approval_required"] = True

### Mock vLLM 补充
新增 RECOMMENDATION_TEMPLATE，含 2~3 条建议示例。
route_to_template 里含 "recommendation" 或 "建议" 时返回此模板。

## 任务四：创建 tests/sprint2_acceptance.py

验收逻辑：
1. 取合成样本中场景A的前5条（已知是连续攻击链）
2. 一起放入 enriched_events 跑完整图
3. 检查 kill_chain["kill_chain_stages"] 长度 >= 3
4. 检查每个 stage 包含 tactic、technique_id、timestamp、description
5. 检查 recommendations 列表长度 >= 1
6. 检查每条 recommendation 包含 action、impact_scope、priority
7. 检查 human_approval_required == True
8. 检查端到端延迟 < 30秒（杀伤链推理允许更长）

输出格式：
── Sprint 2 验收报告 ──────────────────────────────
  ✓/✗  杀伤链阶段数 >= 3          实际: N 阶段
  ✓/✗  每阶段字段完整             
  ✓/✗  响应建议数 >= 1            实际: N 条
  ✓/✗  每条建议含影响范围         
  ✓/✗  human_approval_required=True
  ✓/✗  端到端延迟 < 30s           实际: Xs
  最终结论：Sprint 2 门禁 通过/未通过
