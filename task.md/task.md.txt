## 任务一：创建 deploy/docker-compose.prod.yml

生产部署的 Docker Compose 文件（区别于 Sprint 0 的 mock 版本）。
包含以下服务：

securepilot-api:
  build: .
  ports: ["8080:8080"]
  environment:
    - VLLM_BASE_URL=${VLLM_BASE_URL}
    - KAFKA_BOOTSTRAP_SERVERS=${KAFKA_BOOTSTRAP_SERVERS}
    - KAFKA_MOCK=${KAFKA_MOCK:-false}
    - DINGTALK_WEBHOOK=${DINGTALK_WEBHOOK}
    - DASHBOARD_URL=${DASHBOARD_URL}
    - NO_PROXY=localhost,127.0.0.1,vllm
    - no_proxy=localhost,127.0.0.1,vllm
  volumes:
    - ./data:/app/data
  depends_on: [postgres, qdrant, redis]
  restart: unless-stopped

postgres:
  image: postgres:16-alpine
  environment:
    POSTGRES_DB: securepilot
    POSTGRES_USER: ${DB_USER}
    POSTGRES_PASSWORD: ${DB_PASSWORD}
  volumes: ["postgres_data:/var/lib/postgresql/data"]
  restart: unless-stopped

qdrant:
  image: qdrant/qdrant:v1.9.0
  volumes: ["qdrant_data:/qdrant/storage"]
  restart: unless-stopped

redis:
  image: redis:7-alpine
  restart: unless-stopped

volumes:
  postgres_data:
  qdrant_data:

## 任务三：创建 deploy/setup.py（一键部署脚本）

一个 Python 脚本，客户运行后自动完成：

### 步骤1：环境检查
检查以下命令是否可用：
- python --version（需要 3.11+）
- docker --version
- docker compose version
任何一个不满足就打印错误并退出。

### 步骤2：创建 .env 文件
如果 .env 不存在，从 .env.example 复制一份。
如果 .env 已存在，跳过并提示"已存在，跳过"。

### 步骤3：创建数据目录
mkdir -p data

### 步骤4：安装 Python 依赖
pip install -r requirements.txt -q

### 步骤5：询问部署模式
打印：
"请选择部署模式："
"[1] Mock 模式（无需 GPU，用于演示）"
"[2] 生产模式（需要已部署的 vLLM 服务）"

## 任务五：创建 demo/inject_demo_data.py（演示数据注入工具）

向系统注入一批演示用的攻击场景数据，用于客户演示。

实现逻辑：
1. 从 tests/fixtures/synthetic_alerts.py 读取 ALL_SAMPLES
2. 按场景分三批注入（场景A、B、C各一批）
3. 每批调用 POST http://localhost:8080/api/trigger
   传入该场景的全部告警
4. 每批注入后等待 20 秒让分析完成
5. 打印进度：
   "注入场景A（APT鱼叉钓鱼）... 完成，thread_id=xxx"
   "注入场景B（勒索软件扩散）... 完成，thread_id=xxx"
   "注入场景C（供应链攻击）... 完成，thread_id=xxx"
6. 最后调用 GET /api/events 打印：
   "演示数据注入完成，共 N 条事件可在 Dashboard 查看"
7. 记得在文件顶部设置：
   import os
   os.environ["NO_PROXY"] = "localhost,127.0.0.1"
   os.environ["no_proxy"] = "localhost,127.0.0.1"
